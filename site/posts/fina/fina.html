1.2 Sharpe

1.2.1 Modeling

收益就是股票价格波动的比例，而风险就是收益数列的方差；

\[ \begin{aligned} \mu_i &= \frac{p_t - p_{t-1}}{p_{t-1}} \\ \sigma_i &= \sqrt{\frac{1}{n-1} \sum_{i=1}^{n} (\mu_i - \bar{\mu})^2} \end{aligned} \]

假如单个股票的收益为 \(E(R_i)\)，权重为 \(w_i\)，共有 \(n\) 支股票；那么投资组合的收益 \(\mu_p\) 和方差 \(\sigma_p\) 就是所有单个股票对应的加权均值；

\[ \begin{aligned} \mu_p &= \sum_{i=1}^n w_i E(R_i) \\ \sigma^2_p &= \sum_{i=1}^{n} w_i^2 \sigma^2_i + \sum_{i=1}^{n} \sum_{j \neq i}^n w_i w_j \sigma_{ij} \end{aligned} \]

投资最优就简化成以下两个殊途同归的方向，要么在保持低风险的同时尽量追求高收益，要么在保持高收益的同时尽量压缩风险。但是无论哪个方向，它们最终都是落地到同一个数值上，即求解最大的 Sharpe-Ratio：

\[ \text{Max Sharpe} = \frac{\mu_p - R_f}{\sigma_p} \]

我们根据当前数据，进行投资预测。

us_stocks_aggr = (
    us_stocks %>%

    summarise(
        mean_returns = mean(returns), 
        stdv_returns = SD(returns)
    )

# convert long to wide to xts
us_stocks_xts = (
    us_stocks %>% 
    pivot_wider(
        names_from = symbol, 
        values_from = returns
    ) %>% 
    as.xts()
)


# create a portfolio specification with constraints and objectives
portfolio = (
    portfolio.spec(assets = colnames(us_stocks_xts)) %>%
    add.constraint(type = "full_investment") %>%
    add.constraint(type = "long_only") %>%
    add.objective(type = "risk", name = "StdDev") %>%
    add.objective(type = "return", name = "mean") %>% 
    add.constraint(type = "box", min = 0, max = 0.3)
)


# fit the Mean-Var-Opt model
portfolio_opt = optimize.portfolio(
    R = us_stocks_xts,
    portfolio = portfolio,
    optimize_method = "ROI",
    trace = TRUE
)


# show optimized weights
(
    tibble(
        symbol = portfolio_opt %>% extractWeights() %>% names(),
        weights = portfolio_opt %>% extractWeights()
    ) %>% 
    filter(weights > 0.05) %>%
    datasummary_df()
)

1.3 GARCH

1.3.1 Modeling

上一节讨论的 ARIMA 模型处理的数据是 stationary 且跟踪样本的均值，这个模型比较适用于除了金融数据以外的宏观层面的具有一定周期性的数据；但是如果一个数据是金融类的、微观的、不具有显著周期性的，那是否有模型可以进行一定程度的预测？这就是我们本节进行讨论的 GARCH 模型，它对比 ARIMA 有非常重要的区别，首先 GARCH 模型处理的数据是不平稳的 non-stationary 即意味着它在某种程度上不符合正态分布，或者说不严格符合正态分布，其次它跟踪的对象是样本的方差，而不是均值，即意味着它更加看重样本的波动性，而不是平均值的高低大小。

GARCH 计算理论很深奥且仅限于金融工程专业，简要过一遍模型即可，不对其进行深入讨论了；GARCH(p, q) 受限于两个组成部分：AR(p)和MA(q)。

计算整个序列的方差：

\[ \text{Base} = Var(y_t) = \alpha_0 \]

计算过去某些期数的AR数值：

\[ AR(p) = \sum_{i = 1}^p \alpha_i e_{t - 1}^2 \]

其中 \(\alpha_1\) 是过去方差对当前方差的短期冲击效应，理论上，这个数值越小越好。

计算往期数据的 MA 数值：

\[ MA(q) = \sum_{j = 1}^q \beta_j \sigma_{t - j}^2 \]

其中 \(\beta_1\) 是过去方差对未来方差的长期冲击效应，理论上，这是个数值是越大越好。

将所有部分相加，即可得到GARCH模型：

\[ \begin{align} \text{Dif(d)} &= \text{Base} + AR(p) + MA(q) \\ \sigma_t^2 &= \alpha_0 + \sum_{i=1}^{q} \beta_i \epsilon_{t-i}^2 + \sum_{j=1}^{p} \gamma_j \sigma_{t-j}^2 \end{align} \]

在数学公式的基础上，结合前文已经整理的数据组，对 MSFT 单只股票进行 GARCH 建模。

# plot by ggplot2
## extract CI to tibble format
garch_msft_pred_tidy = (
    garch_msft_pred %>% 
    pluck("distribution") %>% 
    tsconvert() %>% 
    as_tibble() %>% 
    mutate(
        date = ymd(index), 
        pred_value = as.numeric(value)
    ) %>% 
    select(date, pred_value) %>% 
    group_by(date) %>% 
    summarize(
        pred_mean = mean(pred_value),
        pred_lo = quantile(pred_value, 0.1),
        pred_hi = quantile(pred_value, 0.9)
    )
)


## plot prediction on conditional variance
(
    ggplot() + 
    geom_point(
        data = us_stocks %>% filter(symbol == "MSFT"), 
        aes(x = ymd(date), y = returns),
        color = "gray",
    ) +
    geom_line(
        data = garch_msft_pred_tidy,
        aes(x = date, y = pred_mean), 
        color = "orange", linewidth = 2
    ) + 
    geom_ribbon(
        data = garch_msft_pred_tidy,
        aes(x = date, ymin = pred_lo, ymax = pred_hi), 
        fill = "gray", alpha = 0.3
    ) +
    labs(x = "", y = "", title = "GARCH Prediction of MSFT") +
    theme_light() +
    scale_x_date(date_labels = "%Y-%m")
)
